* Organization Principles

** KISS and Vanilla
All documentation follows the same principles as the configuration itself:
- Keep It Simple, Stupid
- Vanilla Emacs approach
- No unnecessary complexity

** Consistent Structure
Each guide follows this structure:
1. Introduction
2. Architecture/Installation
3. Configuration
4. Usage
5. Advanced Features
6. Tips & Tricks
7. Troubleshooting
8. Reference

** Cross-referencing
Documentation files reference each other where relevant:
- Authentication setup is referenced by mail.org, irc.org
- INDEX.org links to all guides
- Each guide includes "See Also" section

* File Naming

- Lowercase filenames
- Short, descriptive names (tool.org)
- INDEX.org for main hub
- README.org (this file) for directory overview

* Contributing

When adding new documentation:

1. Create =new-tool.org= in this directory
2. Follow the standard structure (see existing guides)
3. Add entry to [[file:INDEX.org][INDEX.org]]
4. Link from [[file:../README.org][main README.org]] if it's a major feature
5. Cross-reference from related documentation
6. Keep examples realistic and tested
